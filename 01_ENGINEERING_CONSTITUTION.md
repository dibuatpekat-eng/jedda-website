# JEDDA Engineering Constitution

## Mission
Build a premium WooCommerce experience that reflects the JEDDA brand.

## Golden Principle
Think like an engineer.
Decide like an owner.
Review like a customer.

## Core Rules
- Never modify Production without approval.
- Always prefer Staging.
- Never assume; verify first.
- Mark findings as VERIFIED / NOT VERIFIED / ASSUMPTION.
- If a decision is low-risk, reversible and high-confidence, execute it.
- Ask only for production, payment, security, infrastructure, customer data or business decisions.
- Recommend one preferred solution before asking.

## Quality Standard
Working ≠ Finished.

Finished means:
- Works correctly
- Feels premium
- Looks refined
- Is maintainable
- Has been tested

## Customer Experience
Review every UI like an extremely detail-oriented premium customer.
Notice typography, spacing, alignment, hierarchy, transitions, loading, responsiveness and checkout friction.

## Time
Estimate scope first.
Small: ~5 min
Medium: ~15 min
Large: break into phases and report progress.

## Git
Keep commits small.
Never commit secrets or credentials.
Never modify WordPress/WooCommerce core directly.

Optimize for:
Quality × Speed × Ownership
